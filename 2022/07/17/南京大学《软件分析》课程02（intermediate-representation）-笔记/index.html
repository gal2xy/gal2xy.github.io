南京大学《软件分析》课程02（Intermediate Representation）——笔记

Compilers and Static Analyzers

Compiler

将源代码编译成机器代码（二进制码），并进行报错。

过程：

  1. 词法分析

    词法分析器(Scanner)根据规则（由Regular Expression描述），若全合法则生成Tokens，交给语法分析器(Parser)。

  2. 语法分析

    语法分析器(Parser)，根据语法规则（由Context-Free Grammar描述），将程序转换成AST。

    注：

    为什么用Context-Free Grammar而不用context-sensitive grammar？

    答：Context-Free Grammar对语言语法的表达能力弱。对于编程语言的语法分析，追求速度，Context-Free Grammar更适合。

  3. 语义分析

    类型检查器(Type Checker)，由Attribute Grammar描述规则。

    例：

    1. str类型值与int类型值进行运算会发生报错。

  4. 优化

    通过转换器转换成IR（中间态），进行程序分析、代码优化。

  5. 机器代码生成

以IR为界，IR之上的成为前端，IR之后的成为后端。

AST vs IR

  • AST

    抽象语法树，是一种中间表示结构。

  • IR

    三地址码，也是一种中间表示结构。（形式有点类似汇编）

这些特性使得IR作为静态分析器更合适。

IR: Three-Address Code(3AC)

  • 三地址码

    • There is at most one operator on the right side of an instruction.

    • Each 3AC contains at most 3 addresses.Address can be one of the following:

      • Name: a, b

      • Constant: 3

      • Compiler-generated temporary: t1, t2

    • Each type of instructions has its own 3AC form

    • Some Common 3AC Forms:

      • x = y bop z

      • x = uop y

      • x = y

      • goto L

      • if x goto L

      • if x rop y goto L

    x,y,z: addresses
    bop: binary arithmetic or logical operation
    uop: unary operation(minus,negation,casting)
    L: a label to represent a program location
    rop: relational operator(>,<,==,>=,<=,etc)
    goto L: unconnditional jump
    if……goto L: conditional jump

    例如：

    t2 = a + b + 3，显然不符合三地址码规则，所以进行更改，引入临时变量t1：

    t1 = a + b
    t2 = t1 + 3

概念

  • All assignments in SSA are to variables with distinct names

    • Give each definition a fresh name

    • Propagate fresh name to subsqeuent uses

    • Every variable has exactly one definition

    ​\( \emptyset(x_0.x_1) \)​has the value ​\( x_0 \)​ if the control flow passes through the true part of the conditional and the value ​\( x_1 \)​ otherwise.（真就​\( x_0 \)​，假就​\( x_1 \)​）

优缺点

  • 优点

    • 程序流中的信息被间接指定唯一变量名，可以帮助一些更简单的分析，如flow-insensitive analysis（速度快但不精确）可以获得flow-sensitive analysis的效果（速度低但精确），也就是说更精确了。

  • 缺点

    • SSA引入太多变量和phi-function（分支过多造成的）。

    • SSA转换成机器代码的效率低（个人猜测是由上一点造成的）

Basic Blocks(BB)

概念

Basic blocks (BB) are maximal sqeuences of consecutive three-address instructions with the properties that

  • It can be enterd only at the begining（the first instruction in the block）

  • It can be exited only at the end（the last instruction in the block）

例如：

根据下列三地址码指令构建Basic Blocks

(1) x = input
(2) y = x-1
(3) z = x*y
(4) if z < x goto (7)
(5) p = x / y
(6) q = p + y
(7) a = q
(8) b = x + a
(9) c = 2a - b
(10) if p == q goto (12)
(11) goto (3)
(12) return

首先(1)是作为BB1入口的；(2)因不作为其他指令的入口和出口，可以融入BB1；再看(3)，由于(11)会跳转到(3)，倘若将(3)融入BB1，就会出现两个不同的入口，不满足BB定义，故BB1含(1)(2)。

同理，BB2 = (3)(4)，BB3 = (5)(6)，BB4 = (7)(8)(9)(10)，BB5 = (11)，BB6 = (12)。

Basic Blocks划分规则

1）Determine the leaders in P

  • The first instruction in P is a leader

  • Any target instruction of a conditional or unconditional jump is a leader

  • Any instruction that immediately follows a conditional or unconditional jump is a leader

2）Build BBs for P

  • A BB consists of a leader and all its subsequent instructions until the next leader

Control Flow Graphs(CFG)

概念

将结点用箭头连接的控制流图。这些结点是Basic Block。

如何构造

1）加边规则

  • There is an edge from block A to block B if and only if

    • There is a conditional or unconditional jump from the end of A to the begining of B.

    • B immediately follows A in the original order of instructions and A does not end in an unconditional jump.

2）更改跳转指令

  • It is normal to replace the jumps to instruction labels by jumps to basic blocks.

    这样可以减少程序改变造成跳转标签改变带来的影响。

3）添加进入块和退出块

  • Usually we add two nodes, Entry and Exit.

    • They do not correspond to executable IR.

    • A edge from Entry to the BB containing the first instruction of IR.

    • A edge to Exit from any BB containing an instruction that could be the last instruction of IR.

前驱、后继

块之间的关系。比如A，B之间有边，并且方向由A指向B，则称A是B的前驱，B是A的后继。一个块可以有多个前驱、多个后继。
