Preliminaries Definitions Analysis

Input and Output States

  • Each execution of an IR statement transforms an input state to a new output state.

  • The input (output) state is associated with the program point before (after) the statement.

In each data-flow analysis application, we associate with every program point a data-flow value that represents an abstraction of the set of all possible program states that can be observed for that point.

Data-flow analysis is to find a solution to a set of safe-approximation directed constraints on the IN[s]‘s and OUT[s]‘s, for all statements.

  • constraints based on semantics of statements( transfer functions )

  • constraints based on the flows of control

Reaching Definitions Analysis

Reaching Definitions

A definition d at program point p reaches a point q if there is a path from p to q such that d is not “killed” along that path（定义d的程序点p能到达q的条件是p到q的路径上没有重新定义d的语句）

  • A definition of a variable v is a statement that assigns a value to v（定义v的地方是给v赋值的语句）

  • Translated as : definition of variable v at program point p reaches point q if there is a path from p to q such that no new definition of v appears on that path

Understanding Reaching Definitions

  • Data Flow Values/Facts

    • The definitions of all the variables in a program

    • Can be represented by bit vectors

    即，数据流值包含程序中所有被定义的变量，并且可以用比特向量表示这些变量。

  • Transfer Function and Control Flow

    e.g.

    D: v = x op y

    This statement “generates” a definition D of variable v and “kills” all the other definitions in the program that define variable v, while leaving the remaining incoming definitions unaffected.

    (D处的输出OUT[D]为D处生成的变量v，以及输入变量中除程序中其他定义v的地方)

Algorithm of Reaching Definitions Analysis

why it can finally stop?

  1. 程序不变，gen和kill不变。

  2. 当IN改变时，对应的改变位要么被kill，要么保留。

  3. 被kill的话，OUT不变，那么其后的OUT 也不会变，即使有到达前面的路径edge(即存在循环)，也不会影响IN，于是这个算法就停下来了。

  4. 被保留的话，假设有某位从0->1，（不存在循环容易知道算法会停止）若存在循环，在该循环路径中，分两种情况：

    ①不存在kill该位的basic block，那么会一直保留，经过多次循环，算法停止运行；

    ②若存在kill该位，但又有gen该位的（因为0->1说明必有gen它的地方，且kill在gen前），无论是kill或是gen，其对应的输出OUT不变，而对于在该循环路径中的其他对该位无影响的basic blocks，保留该位。于是经过几次循环，算法停止。

  5. 被保留的话，假设有某位从1->0，若存在循环，在该循环路径中，分两种情况：

    ①只存在kill该位的basic block，照应4.①的分析，算法停止。

    ②既存在gen又存在kill该位的（gen在kill前），照应4.②的分析，算法停止。

挺好奇真的有(5)这样的情况吗(1->0)？分析了一下好像并没有：

假设这样的改变发生在basic block B 的输入处，因为是从1->0，所以其输入（多个或一个）中必定有一个（假设OUT[S]）中的第i位为1，那么无论经过多少次循环，IN[B]都不可能存在1->0的情况。倘若说OUT[S]会随其他basic blocks的OUT的影响，那么算法执行的最初阶段OUT[A]就只能是全0，就不可能有第i位为1的情况。

故每次迭代的OUT必定大于或等于上一次的OUT，而程序本身大小是有限制的，于是比特向量有范围，随着算法的不断执行，就会达到稳定状态（不动点），算法因此停止下来。
