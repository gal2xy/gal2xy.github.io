Hexo博客搭建失败经历

个人觉得Wordpress不适合本人博客搭建的要求，所以尝试在轻量应用服务器上重装博客系统。经历过三天三夜的折磨，我放弃了，还是wordpress更适合新手。

但这次经历让我学习到了不少东西。所以就将其记录下来。

hexo指令不存在

在安装hexo-cli后，执行hexo init命令无效，显示

-bash: hexo: command not found

在查询了他人博客后（使用命令npm install -g hexo-cli 安装Hexo报错bash: hexo: command not found问题解决），找到了方法，使用命令如下：

npx hexo init

在大牛阮一峰的网络日志中，他是这么描述的：“npx想要解决的主要问题，就是调用项目内部安装的模块”，所以可以理解为在命令行下调用，可以让项目内部安装的模块用起来更方便，npx运行的时候，会到node_modules/.bin路径和环境变量$PATH里面，检查命令是否存在，所以系统命令也可以调用，即上面的命令安装不成功的时候加上npx的话也许就可以成功了。（from 缓月）

 

npx和npm的区别

npx用来解决全局命令行工具只能有一个的问题。
​
比如装个webpack,使用的是4.x，可是已经装了全局的1.x版本并且还要继续使用，这个时候可以不装在全局，用npx webpack代替webpack命令，互不干扰。
​
并且推荐这么使用。

Permission denied (publickey,gssapi-keyex,gssapi-with-mic)

在本地测试git仓库是否可用，另找空白文件夹，执行下列命令

git clone git@server_ip:/home/git/repos/blog.git#(server_ip为自己买的网站)

很可惜，当我执行的时候，并没有成功，显示Permission denied (publickey,gssapi-keyex,gssapi-with-mic)

经查询，需要修改/ect/ssh/sshd_config配置文件，修改如下：

- PermitRootLogin yes
- RSAAuthentication yes
- PubkeyAuthentication yes
- AuthorizedKeysFile .ssh/authorized_keys
- PasswordAuthentication yes
- UsePAM yes

没有的可以自己加上，被注释删去注释符就行了。

出现如下情况就说明clone成功了（虽然是个空仓库）。

err: YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key (8:1)

在本地使用npm start调试

npm run deploy

不过出现了这样的错误

err: YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key (8:1)

报错信息是提示冒号后面少了空格，只需要根据提示的错误位置补上空格即可。

修改sshd_config再重装系统，使用远程连接出现Permission denied错误

由于上面操作使得页面连接不了，于是再次重装，然而发现远程连接都不可以了，原因是在解决Permission denied (publickey,gssapi-keyex,gssapi-with-mic)问题时修改PasswordAuthentication yes，而重装的文件里面是注释掉的。

#报错信息
All configured authentication methods failed

根据腾讯云Cloud-Init 和 Cloudbase-Init 问题（云服务器 Cloud-Init 和 Cloudbase-Init 问题），在服务器的执行命令界面执行下列命令：

rm -rf /var/lib/cloud #删除 cloud-init 缓存目录
/usr/bin/cloud-init init –local #执行完整的 cloud-init 初始化
/usr/bin/cloud-init init #根据配置的数据源拉取数据
/usr/bin/cloud-init modules –mode=config #指定运行 config stage
/usr/bin/cloud-init modules –mode=final #指定运行 final stage

记得把文件配置修改回去。

nginx无法启动

执行命令

systemctl start nginx

报错

Job for nginx.service failed because the control process exited with error code. See “systemctl status nginx.service” and “journalctl -xe” for details.

查询得知，

  1. 先检查nginx配置文件正否正确。输入nginx -t 命令，如果反回 successful表示配置文件无错误，否则说明配置文件有错误。

  2. 我这里是没问题的，如果有问题参考 Nginx——Nginx启动报错Job for nginx.service failed because the control process exited with error code

  3. 查看nginx服务状态，根据服务状态去判断报错原因

    systemctl status nginx.service

    我这里的情况显示如下

    跟博主的情况不一样诶。尝试其他方法。

    通过命令ps -ef grep nginx查询nginx主进程号

    需要kill掉重复的进程，我这里执行的是kill 10190（根据实际情况定）。重启nginx后成功了。

没有找到站点

到宝塔界面的做菜单栏的网站中添加域名，我这里添加的是Node项目。

502 bad gateway

我这里的原因是因为修改了博客主题而导致的，而这个问题的原因有很多种，可偏偏找不到我这种的。尝试重装多次，更换主题仍会导致这样的结果。于是放弃了。
